## Thread Structure (5-7 tweets)

### Tweet 1 (Hook)

The single most compelling insight, number, or decision. Must stop scrolling. Max 280 chars. No hashtags in tweet 1.

### Tweets 2-5 (Body)

One point per tweet. Specific details: numbers, screenshots, decisions, what failed. Each tweet must read standalone but link forward.

Include `[ATTACH: description of screenshot/GIF]` placeholders where visual evidence would strengthen the point.

### Tweet N (Close)

Takeaway + soft CTA. Options:
- "Follow for more of the build"
- "Building this at @exampilot -- follow along"
- A genuine question to spark replies

Never: "check out our product" or any hard CTA.

## Rules

- 5-7 tweets max. Shorter threads outperform longer ones.
- No em dashes. Use commas, periods, or colons.
- No banned phrases (see `marketing/context/content-standards.md`).
- Every thread must contain at least one specific number.
- Screenshots/GIFs where possible (3x engagement vs text-only).
- Author voice: match `references/voice-{author}.md`, NOT voice-house.md.
- No links in any tweet except self-reply. X algorithm deprioritises link tweets.
- Number tweets as 1/N format.

## Output

Save to: `build-in-public/pipelines/outreach/x-thread-{author}-{slug}-YYYY-MM-DD.md`
